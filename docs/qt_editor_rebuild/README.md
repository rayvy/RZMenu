# qt_editor_rebuild

Это рабочая папка для выноса `qt_editor` из Blender в отдельный модуль.

Цель:
- убрать прямую зависимость UI от `bpy`
- оставить Blender как тонкий адаптер
- сделать ядро редактора переиспользуемым вне Blender
- подготовить путь к C++-версии с GPU-рендером и отдельным мостом

Содержимое:
- `current_state.md` - что сейчас есть и почему это неудобно
- `target_architecture.md` - целевая структура модулей
- `migration_plan.md` - как выносить по шагам без большого взрыва

Базовый принцип:
- `qt_editor` не должен знать о Blender больше, чем нужно для обмена данными
- `BlenderBridge` должен стать узким API-слоем, а не местом, где живёт логика
