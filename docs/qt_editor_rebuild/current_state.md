# Текущее состояние `qt_editor`

Коротко: сейчас `qt_editor` - это не отдельный редактор, а Blender-встроенный PySide6-интерфейс.

Что видно в коде:
- `RZMenu/__init__.py` подключает `qt_editor` только если найден `PySide6`
- `qt_editor/__init__.py` создаёт Blender-оператор запуска окна
- `qt_editor/core/launcher.py` держит Qt внутри процесса Blender через timers и handlers
- `qt_editor/core/blender_bridge.py` содержит вызовы `bpy`, `undo_push`, импорт/экспорт шаблонов и refresh UI
- многие виджеты напрямую импортируют `bpy`, а не работают через абстракцию

Почему это ломает архитектуру:
- логика редактора привязана к версии Blender и его API
- UI и бизнес-логика живут в одном процессе с Blender
- любой тяжёлый цикл или лишний callback бьёт по главному потоку
- переиспользовать редактор отдельно от Blender почти невозможно

Суть ошибки векторa:
- сначала был построен Blender-аддон
- потом в него втащили полноценный редактор
- в итоге получился не модуль, а сцепка из UI, логики и Blender-specific кода

Главный вывод:
- `qt_editor` надо разрезать на независимое ядро, UI и адаптер к Blender
