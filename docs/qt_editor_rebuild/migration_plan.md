# План выноса `qt_editor`

## Этап 1. Зафиксировать границы

- отделить `core model` от Blender-обвязки
- сделать список того, что сейчас импортирует `bpy`
- пометить всё, что можно вызвать только внутри Blender

## Этап 2. Убрать прямой `bpy` из ядра

- оставить `bpy` только в адаптере
- вместо прямых вызовов сделать интерфейс команд
- чтение данных перевести в snapshot-модель

## Этап 3. Выделить транспорт

- выбрать способ связи:
  - локальный socket
  - JSON-RPC
  - named pipe
- на этом уровне не должно быть ни Qt, ни Blender-specific логики

## Этап 4. Собрать standalone app

- отдельный процесс редактора
- запуск через аддон или вручную
- синхронизация с Blender по протоколу

## Этап 5. Подключить GPU viewport

- сначала скелет сцены и примитивы
- потом интерактивный выбор и трансформации
- потом текстуры, слои, эффекты и спец-инструменты

## Практический приоритет

1. Стабильный core
2. Протокол обмена
3. Отдельное окно редактора
4. GPU viewport
5. Vulkan backend как следующий шаг, если нужен
